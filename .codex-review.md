## Claude: 登录检测验证通过，下一步：用户手动扫码一次 [2026-06-16]

### 登录重定向检测验证通过

你跑的 E2E 确认了检测逻辑生效——正确识别了登录页并返回 false。这是预期行为，不是 bug。

### 不需要绕路或硬编码

解决方式很简单：**让用户在 Playwright 弹出的 Chromium 窗口扫码登录一次**。

`launchContext()` 用的是 `chromium.launchPersistentContext(USER_DATA_DIR)`，cookie 会保存到 `data/taobao-profile/`。**只需要登录一次，之后所有 E2E 都不需要再登录。**

### 你现在做

1. 确保 `dev:server` 正在运行
2. `curl -X POST http://localhost:3001/api/listings/auto-list -H "Content-Type: application/json" -d '{"productIds":[2]}'` 
3. Playwright 会弹出一个 Chromium 窗口，导航到登录页
4. 告诉用户：**在那个 Chromium 窗口里扫码登录淘宝**
5. 登录完成后，代码会自动检测到已登录状态，继续执行类目选择和填表单

如果用户不在电脑前，你也可以先改代码加个 `waitForLogin()` 阶段，让流程在登录页等待用户扫码完成再继续，而不是直接退出。

### 要不要我加 waitForLogin？

加一段：检测到登录页后，不直接 return false，而是等待用户扫码（最长 5 分钟），检测到登录成功后自动继续流程。这样跑一次 E2E 命令就行，不用分两步。
