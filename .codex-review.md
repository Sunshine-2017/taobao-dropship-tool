### 二、核心阻塞问题

**fillPrice() / fillStock() 在淘宝新版发布页找不到"一口价"/"库存"输入框**

前次E2E运行日志（product 3）：
`
Form: title=true price=false qty=false desc=true brand=false packaging=false origin=false freight=true images=false
提交错误: "表单校验: 必填项不能为空; 1:1主图不能为空; 必填项未填"
`

**根因分析**

代码中有两个版本的 taobao-auto-list.js，内容不同：

1. **dist 版本**（实际运行版本）— fillPrice 逻辑:
   - 遍历所有 input
   - 对每个 input 通过 inp.closest('[class*="form"],...') 或 parentElement?.parentElement 获取父容器
   - 检查 parentText 是否包含 '一口价'（编码存疑: '涓€鍙ｄ环'）
   - 回退用 Playwright locator 再走一层父文本匹配

2. **src 版本** — fillPrice 逻辑:
   - 先尝试 document.querySelector('#sell-field-price') + 内部 input
   - 回退同样遍历 input + 父文本匹配

**两个版本都不工作的可能原因**:
- 淘宝新版发布页 DOM 结构变了
- "一口价"/"总库存"文案不在 input 的父链（closest/form/item/field）中
- 文案可能在更远的兄弟元素、label 或远处的 fieldset 里
- 输入框可能是 Shadow DOM 下的 custom element
- ID #sell-field-price 和 #sell-field-quantity 可能不存在了

## Claude: 搜索问题的真正原因 [2026-06-18]

### 终于找到原因了

**不是搜索没触发，是搜索结果展示在别的地方！**

每次 E2E 输出 `Found 34 category results`，但仔细看这 34 个结果的内容——全是"蓝牙耳机"、"洗发水"、"现代文学"这些东西。这明显是**默认类目树**，不是搜索结果。

搜索框确实填入了"花茶"（placeholder="可输入产品名称、类目关键词、条码信息"），Enter 也按了。但淘宝的搜索结果是显示在**输入框下方的下拉菜单/浮层**里的，而不是替换掉页面上的默认类目树。

我们现用的选择器 `.sell-rich-text, [class*="category"], [class*="result"], ...` 捕获的是树形目录里的节点，搜不搜索都在那。

### 怎么修

搜索后需要找**下拉菜单/Dropdown/Popover**里的元素，而不是树里的。比如：
- `[class*="dropdown"]`
- `[class*="popup"]`
- `[class*="menu"]`
- `.ant-select-dropdown`
- `.next-overlay-wrapper`
